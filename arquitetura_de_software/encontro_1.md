## Competências
- Compreender as diferenças entre os níveis de abstração do modelo de análise e projeto para elaborar arquitetura de software
- Escolher a arquitetura adequada para resolver problemas de software
- Analisar padrões de projeto de software (Design Patterns) e como eles se aplicam no desenvolvimento de software
- Aplicar padrões de arquiteturas corporativas para o desenvolvimento de software
- Desenvolver um DAS (Documento de Arquitetura de Software) para demonstrar de forma abstrata um software

---

## Encontros

### Encontro 1
- Apresentação da Disciplina

---

### Encontro 2
- **Níveis de abstrações arquiteturais**
  - Compreender os conceitos de níveis de abstração
  - Modelo de análise e modelo de projeto para criar modelos que representam a solução de desenvolvimento de software e identificar os diferentes gêneros, estilos e padrões arquiteturais de sistemas

---

### Encontro 3
- **Diferentes arquiteturas para diferentes problemas**
  - Monolíticas
  - Distribuídas
  - Multicamadas (n-Tier)
  - Cliente-servidor
  - Orientadas a serviço (SOA)

---

### Encontro 4
- **Diferentes arquiteturas para diferentes problemas**
  - Microserviços
  - Ponto-a-ponto (P2P)
  - Publish-Subscribe (Pub/Sub)
  - Pipes-and-filters (PF) para o desenvolvimento de software

---

### Encontro 5
- **A importância da prototipação na arquitetura de software**
  - Desenvolver protótipo evolutivo de alta fidelidade de um sistema ou de parte de uma solução de sistema para testar e avaliar a eficácia do design global utilizado para resolver um problema específico

---

### Encontros 6 e 7
- **Padrões arquiteturais de Interface com o Usuário (UI)**
  - MVC - Model-View-Controller
  - MVP - Model-View-Presenter
  - MVVM - Model-View-ViewModel

---

### Encontros 8 e 9
- **Padrões arquiteturais de aplicações corporativas**
  - Analisar os conceitos de arquitetura n-tier para a criação de projetos que utilizem 4 camadas (client-tier, presentation-tier, application-tier, database-tier)
  - Aplicar os conceitos de arquiteturas multicamadas, orientadas a serviço e microserviços para desenvolver aplicações corporativas

---

### Encontros 10, 11 e 12
- **Padrões de projeto (Design Patterns)**
  - Abstract Factory, Builder, Factory Method, Prototype, Decorator, Façade, Flyweight, Proxy, Chain of Responsibility, Command, Interpreter, Iterator, Mediator, Memento, Observer, State, Strategy, Template Method, Visitor, Singleton, Adapter, Bridge, Composite

---

### Encontro 13
- **Padrões de integração de sistemas e computação em nuvem**
  - Utilizar técnicas de integração contínua, mensageria, microsserviços, contêineres para integrar sistemas de computação em nuvem

---

### Encontro 14
- **Documento de Arquitetura de Software (DAS) em Metodologias Ágeis**
  - Desenvolvimento de um DAS para demonstrar as diferentes visões de um sistema: estática, dinâmica, lógica (funcionalidades), processos (comportamento), implementação (desenvolvimento) e implantação (física)
